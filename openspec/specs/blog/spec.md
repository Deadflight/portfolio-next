# Blog Specification

## Purpose

Headless CMS blog with EN/ES i18n via Sanity.io. Server-rendered listing + detail pages with Portable Text content, locale filtering via GROQ, PostCard/PostBody components, and on-demand revalidation through Sanity webhooks.

## Fixes (PR-89 Regressions)

### FR-01: postBySlugQuery locale filter

The `postBySlugQuery` in `sanity/lib/queries.ts` MUST filter by both `slug.current == $slug` AND `locale == $locale`. The listing query `postsByLocaleQuery` already filters by locale — it SHALL NOT change. This fix prevents cross-locale data leaks on the detail page.

| ID | Escenario | Given | When | Then |
|----|-----------|-------|------|------|
| E01 | Match in current locale | Post "hello-world" exists in EN and ES | `fetch(postBySlugQuery, { slug: "hello-world", locale: "en" })` | Returns the EN post |
| E02 | Wrong locale returns null | Post "hola-mundo" exists only in ES | `fetch(postBySlugQuery, { slug: "hola-mundo", locale: "en" })` | Returns `null` |
| E03 | Listing unaffected | Posts in both locales | `fetch(postsByLocaleQuery, { locale: "en" })` | Only EN posts returned |

### FR-02: Sanity client consolidation

Two Sanity clients exist: root `sanity/lib/client.ts` (with stega + draft branching) and `src/sanity/lib/client.ts` (autogenerated by `sanity init`). The system MUST consolidate into root `sanity/lib/client.ts` and remove `src/sanity/lib/client.ts`. The consolidated client MUST:

- Export `getClient()` factory that branches on `draftMode().isEnabled` (CDN in production, token-based direct in draft)
- Work for Server Components (`fetch()` directly)
- Keep `urlFor()` from `src/sanity/lib/image.ts` (image builder depends on `projectId`/`dataset` from `src/sanity/env.ts`)

| ID | Escenario | Given | When | Then |
|----|-----------|-------|------|------|
| E04 | Production fetch | `draftMode().isEnabled === false` | `getClient().fetch(query)` | Uses `useCdn: true`, no token |
| E05 | Draft mode fetch | `draftMode().isEnabled === true` | `getClient().fetch(query)` | Uses `useCdn: false` + `SANITY_API_READ_TOKEN` |
| E06 | No dangling imports | All `.ts/.tsx` files | Grep for `src/sanity/lib/client` | Zero matches |

## Non-Functional

### NFR-01: Performance

- **ISR**: Both listing and detail pages SHALL export `revalidate = 60` for background revalidation
- **Static generation**: Detail page SHALL use `generateStaticParams` to pre-build all known (slug, locale) pairs at build time
- **Image optimization**: Cover images SHALL use `next/image` with Sanity CDN `remotePatterns`; inline Portable Text images SHALL use `urlFor().width(800).auto('format').quality(80)`
- **Bundle**: PostBody SHALL NOT import syntax highlighting libraries (no `highlight.js`, `prism.js`) in this change

### NFR-02: SEO

- Detail page SHALL implement `generateMetadata` returning `title`, `description`, and `openGraph` fields including cover image as `og:image`
- Listing page SHALL set `<title>` to `"Blog | Carlos Correa"`
- All blog routes SHALL respect locale prefix for proper hreflang

### NFR-03: Accessibility

- Cover images SHALL have meaningful `alt` text (post title as fallback)
- Nav blog link SHALL use `next-intl` `<Link>` with proper focus management
- Empty state SHALL be a visible text element, not hidden/aria-only
- Heading anchors in PostBody SHALL use `aria-hidden="true"` for the permalink icon

### NFR-04: Error Handling

| Scenario | Behavior |
|----------|----------|
| Sanity API unavailable | Next.js error boundary catches (existing pattern) |
| `coverImage` undefined | Conditional render — no image, no placeholder |
| `description` undefined | Omit meta description, don't render empty element |
| `tags` undefined/null | Omit tag pills, don't render empty container |
| `publishedAt` undefined | Omit date line entirely |
| Slug + locale mismatch | `notFound()` → blog-specific 404 |
| Webhook secret missing in env | Validate optional at handler; return 401 gracefully |

## Data Contracts

### Sanity Query: `postBySlugQuery` (fixed)

```groq
*[_type == "post" && slug.current == $slug && locale == $locale][0] {
  _id, title, "slug": slug.current,
  description, body, publishedAt, locale, tags,
  "coverImage": coverImage.asset->url
}
```

### Sanity Query: `postsByLocaleQuery` (unchanged)

```groq
*[_type == "post" && locale == $locale && defined(slug.current)]
| order(publishedAt desc) {
  _id, title, "slug": slug.current,
  description, publishedAt, tags,
  "coverImage": coverImage.asset->url
}
```
